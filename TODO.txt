ЛОГИКА РЕАЛЬНОГО БАНКА (справка):
  • Клиент (Client) открывает счёт (Account) в банке
  • Счета бывают: дебетовый (DebitAccount), накопительный (SavingsAccount), кредитный (CreditAccount)
  • Клиент пополняет счёт (deposit), снимает деньги (withdraw), делает переводы (transfer)
  • Каждая операция — это транзакция (Transaction) с датой, суммой, типом
  • На накопительный счёт начисляются проценты (interest rate)
  • Кредитный счёт имеет лимит (credit limit) и долг (debt)
  • Банк хранит историю всех транзакций (аудит/audit)
  • При недостатке средств — отказ с исключением (InsufficientFundsException)
  • Счёт может быть заблокирован (BLOCKED), активен (ACTIVE) или закрыт (CLOSED)

  [ ] Раскомментируй зависимости в pom.xml
  [ ] Создай BankApplication.java с @SpringBootApplication
  [ ] Настрой application.properties (порт, БД)
  [ ] Запусти приложение: mvn spring-boot:run

ШАГ 2 — REST Controller
  [ ] @RestController, @RequestMapping
  [ ] @GetMapping, @PostMapping, @PutMapping, @DeleteMapping
  [ ] @PathVariable, @RequestParam, @RequestBody
  [ ] ResponseEntity<T>
  [ ] ЗАДАНИЕ: AccountController:
        - GET /api/accounts — все счета
        - GET /api/accounts/{id} — счёт по ID
        - POST /api/accounts — создать счёт
        - PUT /api/accounts/{id}/deposit — пополнить
        - PUT /api/accounts/{id}/withdraw — снять
        - POST /api/accounts/transfer — перевод

ШАГ 3 — Service Layer
  [ ] @Service, @Transactional
  [ ] Разделение бизнес-логики и контроллера
  [ ] ЗАДАНИЕ: AccountService с методами бизнес-логики

ШАГ 4 — Repository (JPA)
  [ ] @Entity, @Id, @GeneratedValue, @Column
  [ ] @OneToMany, @ManyToOne, @ManyToMany
  [ ] JpaRepository<T, ID>
  [ ] @Query — кастомные JPQL запросы
  [ ] ЗАДАНИЕ: AccountEntity, ClientEntity, TransactionEntity
  [ ] ЗАДАНИЕ: findByClientId, findByStatusAndBalanceGreaterThan

ШАГ 5 — Dependency Injection
  [ ] @Autowired (через конструктор — лучшая практика!)
  [ ] @Component, @Service, @Repository, @Controller
  [ ] @Bean, @Configuration
  [ ] @Qualifier — выбор конкретной реализации

ШАГ 6 — Валидация
  [ ] @Valid, @NotNull, @NotBlank, @Min, @Max, @Size, @Email
  [ ] BindingResult для обработки ошибок валидации
  [ ] @ExceptionHandler, @ControllerAdvice для глобальной обработки ошибок

ШАГ 7 — Безопасность (Spring Security)
  [ ] @EnableWebSecurity
  [ ] Базовая HTTP аутентификация
  [ ] JWT токены (advanced)
  [ ] @PreAuthorize, @Secured
